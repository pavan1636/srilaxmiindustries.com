# CONCEPT 012 — What is S3?

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
WHAT IS IT?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
S3 (Simple Storage Service) is AWS's unlimited cloud file storage.
Files are stored as objects inside buckets. It is designed for
99.999999999% (11 nines) durability — your files will not be lost.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
SIMPLE ANALOGY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Local hard drive = filing cabinet in your office (limited space, one location)
  S3 = an infinite warehouse with armed guards, fire protection,
  and copies stored in multiple buildings across the city

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
3 KEY FACTS TO REMEMBER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. S3 bucket names must be GLOBALLY unique across all AWS accounts
  2. Buckets should block all public access unless explicitly needed
  3. Versioning keeps previous versions of files (rollback protection)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
IN OUR PROJECT (Sri Laxmi Industries)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  File: terraform/s3.tf
  Bucket 1: srilaxmi-enquiry-uploads-pavan
    Purpose: Stores uploaded CAD drawings and PDF technical specs
    Access: Private only (all public access blocked)
    Upload: Backend streams file buffer directly from Multer to S3
    Path: enquiries/{uuid}.{extension}

  Bucket 2: srilaxmi-db-backups-pavan
    Purpose: Stores nightly compressed PostgreSQL database backups
    Access: Private only
    Versioning: Enabled (keeps backup history)
    Upload: scripts/backup.sh → pg_dump | gzip → aws s3 cp

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
INTERVIEW ANSWER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
"In Sri Laxmi Industries, I configured two private S3 buckets via
Terraform. The upload bucket stores CAD drawings submitted by B2B
customers — the Node.js backend uses the AWS SDK to stream files
directly from Multer memory storage to S3 without writing temporary
files to disk. The backup bucket stores nightly gzipped PostgreSQL
dumps. Both buckets block all public access and use IAM role-based
authentication from the EC2 instance profile."

COMMON MISTAKE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
❌ Making S3 buckets public to serve files
✅ Keep buckets private. Use pre-signed URLs or IAM roles for access.
   Public buckets are a top security risk — data breaches happen daily.
