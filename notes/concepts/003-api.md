# CONCEPT 003 — What is an API

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
WHAT IS IT?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  An API (Application Programming Interface) is a set of rules that
  lets two pieces of software talk to each other. A REST API uses
  HTTP methods (GET, POST, PUT, DELETE) to exchange JSON data.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
THE PROBLEM IT SOLVES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Without an API, the frontend would need direct database access,
  which is insecure and tightly coupled. APIs create a clean contract
  between frontend and backend — if the database schema changes, only
  the backend code needs updating, not every page in the frontend.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
SIMPLE ANALOGY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  An API is like a waiter in a restaurant. You (the frontend) tell the
  waiter your order, the waiter takes it to the kitchen (backend), and
  brings back your food (data). You never walk into the kitchen yourself.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
3 KEY FACTS TO REMEMBER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. REST = Representational State Transfer — uses standard HTTP verbs
  2. GET reads data, POST creates, PUT updates, DELETE removes
  3. APIs return JSON — a lightweight format both humans and code can read

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
IN OUR PROJECT (Sri Laxmi Industries)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Our Express backend exposes a REST API with endpoints for:
  - /api/products — list and manage steel product catalog
  - /api/quotes — dealers request quotes for bulk orders
  - /api/dealers — dealer registration and profile management
  - /api/admin — admin operations protected by API key auth
  All responses are JSON. The Nginx reverse proxy forwards any
  request starting with /api/ to the Express container on port 3000.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
INTERVIEW ANSWER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  "In my Sri Laxmi Industries project, I built a REST API with
  Express. The frontend calls endpoints like GET /api/products and
  POST /api/quotes using fetch(). Nginx proxies /api/ requests to
  the Express container. Admin endpoints are secured with an API
  key passed in the request header — this keeps unauthorized users
  from accessing sensitive operations like GDPR data exports."

COMMON MISTAKE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ❌ Using GET requests for operations that change data (e.g., deleting)
  ✅ Use the correct HTTP verb: GET to read, POST to create, PUT to
     update, DELETE to remove. This makes the API predictable and
     follows REST conventions that other developers expect.
